PROBLEM HISTORY

Issue #1 (2026-03-03 15:00)

Symptom: ls and read commands failing to find files on NFS share
Command: ls -lh /mnt/qnappy/cortex-workspace/research/PRIORITY_21_INTELLIGENCE_OPERATIONS.md
Error: File not reported in listings

Root Cause: Tool working directory mismatch

  • Exec commands were defaulting to /home/chopfner/.openclaw/workspace
  • File paths specified as /mnt/qnappy/cortex-workspace/research/FILE.md
  • edit and read tools may use different internal working directories

Fix:

  1. ALWAYS cd /mnt/qnappy/cortex-workspace/research first
  2. Use relative paths after cd (not absolute paths)
  3. Verify with pwd before file operations

Verification Command:

cd /mnt/qnappy/cortex-workspace/research && pwd && ls FILE.md

Issue #2 (2026-03-03 15:35)

Symptom: edit tool failed with "File not found" despite file existing
Command: edit with full path /mnt/qnappy/cortex-workspace/research/APPENDIX_EPSTEIN_FILES_COMPENDIUM.md
Error: Tool returned file not found

Root Cause:

  • edit tool may have different working directory behavior than exec
  • exec with full path works, but edit with full path fails
  • Tool internal caching or directory resolution

Fix:

  1. Use exec with sed for file edits on NFS share
  2. Avoid edit tool for NFS files
  3. Use write tool for complete file overwrites (if needed)

Working Edit Command (exec + sed):

cd /mnt/qnappy/cortex-workspace/research
sed -i 's/old_text/new_text/' FILE.md

OPERATIONAL RULES — MUST FOLLOW

For File Listings

For File Editing

USE THIS:

AVOID THIS:

content here
EOF"
